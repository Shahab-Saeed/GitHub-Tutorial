    COMMIT

    Whenever we make any change in our repository, then it becomes a commit and this commit will store all the information about the chnages

    Confuring Git

    Means that we will give our account's name and email to tell git on which account we wants to chnage

    Confuring git Commands

    • git config --global user.name "My Name"

    here global means everytime we are making chnanges from the same account

    here we can also used local which means that we are making chnages from the specific account on a specific repo

    • git config --global user.email "Email"

    here we have to enter the email of our account which is on github

    • git config --list

    this will gives all the information about our username and email

    cd

    used to chnage the directory

    whenver we wants to go inside of any folder , we use this command

    cd ..

    used to go back to the previous directory

    whenver we wants to go back to the previous folder , we use this command

    ls

    used to see the list

    whenver we wants to see all the files and the folders inside our main folder we can use this command

    ls -Hidden

    used to see the list of the hidden files also

    ls will not show us the hidden files so, whenver we wants to see all the files + hidden files and the folders inside our main folder we can use this command

    git status

    used to see the status of our code

    if our code is not committed then it will gives an error

    to remove an error first we have to use the "add" command and then "commit" command

    there are mainly four to five types of status

    • Untracked
    when you have created a new file but, you haven't add or committed that
      Modified
    when you have made some changes in the file but, you haven't add or committed that
      Staged
    when you have added the file but, you haven't committed that
      Committed
    when you have added and committed the file
      Unmodified
    when the file is unchanged

    • ADD COMMAND

    git add fileName

    adds new or changed file in your working directory to the git staging area

    We can add all the files and changing using "git add ." command

    Commit COMMAND

    git commit -m "Enter Commit message"

    It is the record of change

    PUSH COMMAND

    git push origin main/branch

    changes we made using "git commit ..." command will be committed but we can't see that on github, so we use push command to also see that changes on our github

    OR
